## 1. 白图与蓝图

在电力行业（尤其变电站二次系统、继电保护、屏柜制造与现场施工）中，「白图」「蓝图」都是工程图纸的俗称，指**图纸所处的阶段、用途和效力**不同，而不单指纸的颜色。

### 1.1 白图是什么

**白图**一般指：

- 设计过程中输出的、**白底**打印或电子版的**非最终正式图**；
- 用于**方案讨论、设计校核、厂家配合、逻辑确认、内部流转**；
- 通常**尚未完成全部会签/批准**，或**不作为唯一施工依据**。

常见类型（二次专业语境下）：

| 类型 | 说明 |
|------|------|
| 原理图 / 逻辑图初稿 | 保护逻辑、联锁关系、信号走向的概念与细节讨论 |
| 屏柜布置初稿 | 屏面布置、端子排分区、设备排列的方案比选 |
| 接线图工作图 | 设计院所与屏柜厂、现场之间核对用的工作底图 |
| 白图会审材料 | 设计交底、专业会签、业主审查用的图面 |

特点概括：

- **迭代快**：随设计深化多次改版；
- **效力弱于正式蓝图**：现场不宜仅凭白图擅自改接线；
- **信息可能不完整**：端子号、电缆编号、设备型号等可能在后续版本才定稿。

在口语里，运行、检修人员有时也把**打印出来的原理图、逻辑图**（白底 A3/A4）统称为「白图」，强调它是**用来讲清楚逻辑与结构**的说明性图纸，与现场正式归档的施工图相对。

### 1.2 蓝图是什么

**蓝图**源于早期氨晒/蓝图复印工艺（白线条、蓝底），现多指：

- **经审核批准的正式设计图纸**（或与其等效的正式电子图 PDF/图档）；
- 作为**制造、采购、施工、安装、竣工归档**的依据；
- 图纸上通常有**设计单位出图章、会签、批准、图号、版本**等正式要素。

在二次系统中，蓝图常见包括：

| 类型 | 说明 |
|------|------|
| 正式原理图 / 正式逻辑图 | 定稿后的保护及自动化逻辑 |
| 屏柜接线图 | 屏内配线、端子排接线关系 |
| 端子排图 | 对外接线端子定义 |
| 电缆联系图 / 材料表 | 与一次、其他屏柜的电缆接口 |
| 竣工图（竣工蓝图） | 现场实际接线与变更确认后的最终归档图 |

特点概括：

- **版本受控**：图号 + 版本号，变更需走正式设计变更流程；
- **现场施工依据**：屏柜厂配线、现场二次施工以蓝图为准；
- **与设备、电缆一一对应**：端子号、回路号、设备编码应与实物一致。

> 注：今天很多单位已不再物理「晒蓝图」，但仍用「蓝图」称呼**正式出图**；电子图档（如 PDF、DWG）在管理意义上等同蓝图。

### 1.3 二者关系对比

```text
设计意图 ──► 白图（多轮讨论、校核、会审） ──► 蓝图（批准定稿） ──► 制造/施工/竣工
                ↑ 可频繁修改                      ↑ 变更需正式程序
```

| 维度 | 白图 | 蓝图 |
|------|------|------|
| 阶段 | 设计过程、方案阶段 | 设计定稿、施工与归档阶段 |
| 主要用途 | 沟通、审查、逻辑确认 | 制造、安装、验收、运维归档 |
| 版本管理 | 相对灵活，常标注「草案」「仅供讨论」 | 严格图号版本管理 |
| 现场效力 | 参考、配合理解，一般不作唯一依据 | 正式依据 |
| 典型读者 | 设计、厂家、调试、培训 | 施工队、屏柜厂、运行、档案 |

### 1.4 与 Zeta 产品的关联（屏柜认知）

本项目中，学员在**屏柜认知**模块接触的「正视图、侧视图、后视图」等条目，属于**面向认知与讲解的展示内容**（类似白图/说明图在培训场景下的抽象），用于建立屏柜空间与结构概念：

- **不等同**于设计院出具的正式蓝图；
- **不替代**现场应以正式图纸和实物为准的运维规则；
- 内容可由业务人员在业务库中维护（图片 + 文字描述），与屏柜系统（`ct-screen`）中的结构化设备/逻辑数据只读关联。

若后续接入真实工程图档，建议区分：

- **培训/认知层**：白图性质的示意与讲解（本系统 `cabinet_display_items`）；
- **工程/运维层**：正式蓝图或等效电子图档（通常来自设计单位或屏柜系统，需版本与权限管理）。
