# ReactFlow DAG 自动布局引擎开发任务

## 项目目标

基于 React + TypeScript + ReactFlow 实现一个 DAG（有向无环图）自动布局渲染引擎。

本项目用于展示：

- 继电保护逻辑图
- 电力试验逻辑图
- 教学逻辑结构图
- 流程逻辑图

用户仅提供 JSON 数据。

系统自动完成：

- 节点分层
- 自动布局
- 自动连线
- Fan-In/Fan-Out 优化
- 交叉优化
- 自动居中显示

不需要：

- 节点拖拽
- 节点编辑
- 连线编辑
- 流程设计器能力

这是一个只读展示型结构图。

# 图结构约束

图保证为 DAG。

即：

```text
Input → Gate → Gate → Output
```

不存在环路。

支持：

## Fan-Out

一个节点连接多个下游节点

```text
InputA
├─ Gate1
├─ Gate2
└─ Gate3
```

---

## Fan-In

多个节点汇聚到同一节点

```text
InputA ─┐
         ├─ Gate1
InputB ─┘
```

---

## 多层逻辑

```text
Input
 ↓
Gate
 ↓
Gate
 ↓
Gate
 ↓
Output
```

# 布局要求

## 1. Input 固定最左

所有 Input 节点必须位于：

```text
Layer 0
```

---

## 2. Output 固定最右

所有 Output 节点必须位于：

```text
最后一层
```

---

## 3. 自动计算层级

规则：

```text
Layer(node)
=
max(parent.layer) + 1
```

例如：

```text
InputA
InputB

↓

Gate1

↓

Gate2

↓

Output
```

结果：

```text
Layer0:
InputA
InputB

Layer1:
Gate1

Layer2:
Gate2

Layer3:
Output
```

---

## 4. 节点不允许重叠

布局参数：

```ts
const NODE_WIDTH = 220;
const NODE_HEIGHT = 80;

const HORIZONTAL_GAP = 350;
const VERTICAL_GAP = 120;
```

必须保证：

```text
任意两个节点矩形区域不能重叠
```

---

## 5. 尽可能减少边交叉

实现：

```text
Barycenter Crossing Reduction
```

算法：

对于每一层：

nodeWeight = 所有父节点Y坐标平均值

按照 weight 排序。

至少执行：

```text
3轮迭代
```

降低边交叉数量。

---

## 6. Fan-Out 优化

对于：

```text
A
├─ B
├─ C
└─ D
```

不要直接生成：

```text
A ───── B
A ───── C
A ───── D
```

优先生成：

```text
A ──┬─ B
    ├─ C
    └─ D
```

实现方式：

允许自动插入隐藏节点：

```ts
{
  id: string;
  type: "__split__";
  hidden: true;
}
```

仅用于布局。

不显示。

---

## 7. Fan-In 优化

对于：

```text
A ─┐
    ├─ X
B ─┘
```

尽量保持汇聚点自然。

避免产生大面积折线交叉。

---

## 8. 连线路由

禁止：

```text
BezierEdge
```

禁止：

```text
SmoothStepEdge
```

优先：

```text
StepEdge
```

或者：

```text
Orthogonal Edge
```

目标效果：

```text
──────┐
      │
      └──────
```

逻辑图可读性更高。

---

## 9. 连线不能穿越节点

节点视为障碍物。

要求：

```text
Edge Path 不允许穿过节点矩形区域
```

如果发生碰撞：

自动绕行。

# 特别说明

这是 DAG 自动布局系统，不是树结构布局系统。

请参考 ELK（Layered Layout）思想实现：

- Layer Assignment
- Crossing Minimization
- Coordinate Assignment
- Orthogonal Routing

不要实现简单树布局。

最终目标是：

在复杂继电保护逻辑图场景下，实现：

- 层级清晰
- 连线整齐
- 节点不重叠
- 连线交叉最少
- Fan-Out/Fan-In 结构可读性高
- 数据变化后自动重新布局
